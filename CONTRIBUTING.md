# 贡献指南

感谢您对本项目的关注！我们欢迎任何形式的贡献。

## 如何贡献

1. Fork 本仓库
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个 Pull Request

## 代码规范

- 使用 Black 进行代码格式化
- 使用 Flake8 进行代码检查
- 编写单元测试覆盖新功能
- 更新相关文档

## 提交信息规范

使用清晰的提交信息：

- `feat: 添加新功能`
- `fix: 修复bug`
- `docs: 更新文档`
- `style: 代码格式调整`
- `refactor: 代码重构`
- `test: 添加测试`
- `chore: 构建/工具链更新`

## 问题报告

如果您发现 bug 或有功能建议，请：

1. 检查是否已有相关 Issue
2. 创建新 Issue 并提供详细信息
3. 包含复现步骤（如果是 bug）

感谢您的贡献！
